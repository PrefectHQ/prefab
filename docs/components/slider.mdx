Sliders let users select a numeric value by dragging a handle along a track. They're the right choice when you want users to explore a continuous range rather than type an exact number -- adjusting volume, setting a price filter, or tuning a threshold. The default range is 0--100, but you can customize the bounds, step size, and even add a second thumb for range selection.

## Basic Usage

At minimum, pass a `value` to set the initial position. The slider renders with a track, a filled region up to the current value, and a draggable thumb. Without a `name` or `on_change`, the slider is interactive in the browser but doesn't communicate its value back to your application state -- you'll want one of those for anything beyond a visual demo.

<ComponentPreview json={{"view":{"type":"Slider","min":0,"max":100,"value":50.0,"disabled":false}}}>
<CodeGroup>
```python Python icon="python"
from prefab_ui.components import Slider

Slider(value=50)
```
```json Protocol icon="brackets-curly"
{
  "view": {
    "type": "Slider",
    "min": 0,
    "max": 100,
    "value": 50.0,
    "disabled": false
  }
}
```
</CodeGroup>
</ComponentPreview>

## Displaying the Value

Give the slider a `name` to automatically bind its current value to state, then reference that state with `{{ }}` interpolation. As the user drags the handle, the bound state updates in real time and any components referencing it re-render instantly. This is the simplest way to show the current slider value -- no action wiring required.

The `| number` pipe strips trailing decimals so the display stays clean. This pattern works with any component that supports interpolation.

<ComponentPreview json={{"view":{"cssClass":"gap-2","type":"Column","children":[{"cssClass":"justify-between","type":"Row","children":[{"type":"Label","text":"Volume"},{"content":"{{ volume | number }}%","bold":true,"type":"Text"}]},{"type":"Slider","min":0,"max":100,"value":75.0,"name":"volume","disabled":false}]},"state":{"volume":75}}}>
<CodeGroup>
```python Python icon="python"
from prefab_ui.app import set_initial_state
from prefab_ui.components import (
    Column,
    Label,
    Row,
    Slider,
    Text,
)

set_initial_state(volume=75)

with Column(gap=2):
    with Row(css_class="justify-between"):
        Label("Volume")
        Text("{{ volume | number }}%", bold=True)
    Slider(value=75, name="volume")
```
```json Protocol icon="brackets-curly"
{
  "view": {
    "cssClass": "gap-2",
    "type": "Column",
    "children": [
      {
        "cssClass": "justify-between",
        "type": "Row",
        "children": [
          {
            "type": "Label",
            "text": "Volume"
          },
          {
            "content": "{{ volume | number }}%",
            "bold": true,
            "type": "Text"
          }
        ]
      },
      {
        "type": "Slider",
        "min": 0,
        "max": 100,
        "value": 75.0,
        "name": "volume",
        "disabled": false
      }
    ]
  },
  "state": {
    "volume": 75
  }
}
```

## Range and Step

Use `min`, `max`, and `step` to constrain the slider to a specific domain. The `step` parameter snaps the thumb to fixed increments as it moves -- here the slider covers 0--10 with 0.5 increments, so the user can only land on values like 3.0, 3.5, 4.0. This is useful for ratings, discrete scales, or any input where arbitrary precision would be confusing.

<ComponentPreview json={{"view":{"cssClass":"gap-2","type":"Column","children":[{"cssClass":"justify-between","type":"Row","children":[{"type":"Label","text":"Rating"},{"content":"{{ rating }}","bold":true,"type":"Text"}]},{"type":"Slider","min":0.0,"max":10.0,"value":5.0,"step":0.5,"name":"rating","disabled":false}]},"state":{"rating":5}}}>
<CodeGroup>
```python Python icon="python"
from prefab_ui.app import set_initial_state
from prefab_ui.components import (

    Row,
    Slider,

set_initial_state(rating=5)

        Label("Rating")
        Text("{{ rating }}", bold=True)
    Slider(value=5, name="rating", min=0, max=10, step=0.5)
```

## With Actions

The `on_change` parameter runs actions when the slider value changes. Use `$event` to reference the current numeric value within the action. This gives you full control over what happens when the slider moves -- update multiple state keys, trigger a tool call, or chain several actions together.

<ComponentPreview json={{"view":{"cssClass":"gap-4","type":"Column","children":[{"type":"Slider","min":0,"max":100,"value":50.0,"disabled":false,"onChange":{"action":"setState","key":"level","value":"{{ $event }}"}},{"cssClass":"text-muted-foreground","content":"{{ level | 'Drag the slider' }}","type":"Text"}]}}}>
<CodeGroup>
```python Python icon="python"
from prefab_ui.components import Column, Slider, Text
from prefab_ui.actions import SetState

with Column(gap=4):
    Slider(
        value=50,
        on_change=SetState("level", "{{ $event }}"),
    )
    Text(
        "{{ level | 'Drag the slider' }}",
        css_class="text-muted-foreground",
    )
```
```json Protocol icon="brackets-curly"
{
  "view": {
    "cssClass": "gap-4",
    "type": "Column",
    "children": [
      {
        "type": "Slider",
        "min": 0,
        "max": 100,
        "value": 50.0,
        "disabled": false,
        "onChange": {
          "action": "setState",
          "key": "level",
          "value": "{{ $event }}"
        }
      },
      {
        "cssClass": "text-muted-foreground",
        "content": "{{ level | 'Drag the slider' }}",
        "type": "Text"
      }
    ]
  }
}
```
</CodeGroup>
</ComponentPreview>

<Tip>
For simply displaying the slider's current value, [auto-binding with `name`](#displaying-the-value) is usually easier. Use `on_change` when you need to run custom logic like triggering a tool call or updating multiple state keys.
</Tip>

## Disabled State

Set `disabled=True` to render the slider in a non-interactive state. The thumb is visible but cannot be dragged. This is useful for showing a value that the user can see but not change -- for example, a computed result or a setting that requires a higher permission level.

<ComponentPreview json={{"view":{"type":"Slider","min":0,"max":100,"value":30.0,"disabled":true}}}>

## Range Slider

Set `range=True` to render two thumbs for selecting a low/high range. Pass `value` as a `[low, high]` list to set the initial bounds. This is the natural choice for filter controls -- price ranges, date spans, or any input where the user is defining a window rather than a single point.

Give the slider a `name` to bind the range to state as an array, then use dot-path interpolation to display the individual values. For a slider named `price_range`, the state value is `[20, 80]`, and you can access the low end with `{{ price_range.0 }}` and the high end with `{{ price_range.1 }}`. Both labels update live as either thumb is dragged.

<ComponentPreview json={{"view":{"cssClass":"gap-2","type":"Column","children":[{"cssClass":"justify-between","type":"Row","children":[{"content":"Min: ${{ price_range.0 | number }}","bold":true,"type":"Text"},{"content":"Max: ${{ price_range.1 | number }}","bold":true,"type":"Text"}]},{"type":"Slider","min":0,"max":100,"value":[20.0,80.0],"range":true,"name":"price_range","disabled":false}]},"state":{"price_range":[20,80]}}}>
<CodeGroup>
```python Python icon="python"
from prefab_ui.app import set_initial_state
from prefab_ui.components import Column, Row, Slider, Text

set_initial_state(price_range=[20, 80])

with Column(gap=2):
    with Row(css_class="justify-between"):
        Text("Min: ${{ price_range.0 | number }}", bold=True)
        Text("Max: ${{ price_range.1 | number }}", bold=True)
    Slider(value=[20, 80], range=True, name="price_range")
```
```json Protocol icon="brackets-curly"

        "cssClass": "justify-between",

            "type": "Text"

        "name": "price_range",

  "state": {
    "price_range": [
      20,
```

### Range with Step

Combine `range=True` with `step` to snap both thumbs to fixed increments. This prevents users from selecting awkward fractional values and keeps the range boundaries clean. Here both thumbs snap to multiples of 5.

<ComponentPreview json={{"view":{"type":"Slider","min":0,"max":100,"value":[25.0,75.0],"step":5.0,"range":true,"disabled":false}}}>

## API Reference

<Card icon="code" title="Slider Parameters">

<ParamField body="value" type="float | list[float] | None" default="None">
  Initial value of the slider. Pass a `[low, high]` list when `range=True`.
</ParamField>

<ParamField body="name" type="str | None" default="None">
  State key for the slider's current value. Set this to bind the slider to state for display or form submission.
</ParamField>

<ParamField body="min" type="float" default="0">
  Minimum value of the range.
</ParamField>

<ParamField body="max" type="float" default="100">
  Maximum value of the range.
</ParamField>

<ParamField body="step" type="float | None" default="None">
  Step increment. If not provided, the slider moves continuously.
</ParamField>

<ParamField body="range" type="bool" default="False">
  Enable two-thumb range selection. When `True`, `value` accepts a `[low, high]` list and `on_change` emits the pair as `[low, high]`.
</ParamField>

<ParamField body="disabled" type="bool" default="False">
  Whether the slider is non-interactive.
</ParamField>

<ParamField body="on_change" type="Action | list[Action] | None" default="None">
  Action(s) to execute when the value changes. `$event` is the current numeric value.
</ParamField>

<ParamField body="css_class" type="str | None" default="None">
  Additional Tailwind CSS classes appended to the component's built-in styles.
</ParamField>
</Card>
